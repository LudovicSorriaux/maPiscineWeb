# PROCÉDURE D'UPLOAD FICHIERS WEB VERS ESP8266

## IMPORTANT - NE JAMAIS UTILISER `gulp upload`

La commande `gulp upload` ne fonctionne pas correctement pour ce projet.

## PROCÉDURE CORRECTE D'UPLOAD

### 1. Générer les fichiers .lgz
```bash
cd /Users/ludovic1/Documents/PlatformIO/Projects/maPiscinev4.5Web-d1_mini
gulp buildfs
```

### 2. Uploader INDIVIDUELLEMENT chaque fichier via curl

**Adresse serveur** : `mapiscine.local`

**Upload main.html.lgz** :
```bash
curl -X POST -F "file=@data/html/main.html.lgz" -F "adminPassword=manager" -F "path=/html" http://mapiscine.local/upload
```

**Upload style.css.lgz** :
```bash
curl -X POST -F "file=@data/html/style.css.lgz" -F "adminPassword=manager" -F "path=/html" http://mapiscine.local/upload
```

**Upload script.js.lgz** :
```bash
curl -X POST -F "file=@data/html/script.js.lgz" -F "adminPassword=manager" -F "path=/html" http://mapiscine.local/upload
```

## Notes
- Toujours utiliser `curl` avec les formulaires multipart
- URL : `http://mapiscine.local/upload`
- Mot de passe admin : `manager`
- Path destination : `/html`
- Fichiers sources : `data/html/*.lgz` (après `gulp buildfs`)
